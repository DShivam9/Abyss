# AbsoluteUI Content Guidelines

## Purpose

This document defines how content should be written across AbsoluteUI.

This includes:

* Website copy
* Documentation
* Component descriptions
* Release notes
* GitHub content
* Social media content
* Future marketing material

The goal is consistency.

Every word should reinforce the AbsoluteUI identity.

---

# Core Philosophy

AbsoluteUI is a design-focused product.

The content should reflect the same qualities as the components.

Content should feel:

* Clear
* Intentional
* Confident
* Refined
* Professional

Not:

* Corporate
* Sales-driven
* Overhyped
* Artificial

---

# Primary Writing Principles

## Principle 01 — Clarity First

Users should understand information immediately.

Prefer:

"Interactive card with layered depth and cursor-aware motion."

Avoid:

"Revolutionary next-generation card experience designed to elevate user engagement."

---

## Principle 02 — Show, Don't Sell

Focus on what the component does.

Do not exaggerate.

Good:

"Creates depth through layered hover interactions."

Bad:

"Transforms ordinary interfaces into unforgettable masterpieces."

---

## Principle 03 — Be Specific

Specific descriptions build trust.

Good:

"Parallax image component with smooth scroll-driven motion."

Bad:

"Beautiful image experience."

---

## Principle 04 — Respect the User

Assume the reader is intelligent.

Avoid:

* Excessive explanations
* Clickbait language
* Marketing fluff

Provide useful information.

---

# Voice & Tone

## Voice

AbsoluteUI should sound like:

* An experienced designer
* A skilled frontend developer
* A thoughtful product creator

Not:

* A salesperson
* A startup founder pitching investors
* A social media influencer

---

## Tone

The tone should remain:

* Professional
* Minimal
* Confident
* Helpful

Never:

* Aggressive
* Arrogant
* Overly casual

---

# Writing Hierarchy

Every page should follow:

## Level 1

Headline

Clear purpose.

---

## Level 2

Description

Explain value.

---

## Level 3

Details

Installation.
Customization.
Usage.

---

Users should never be forced to read long blocks of text before understanding a component.

---

# Homepage Copy Guidelines

Homepage content should:

* Explain quickly
* Inspire confidence
* Demonstrate quality

Avoid:

* Long paragraphs
* Buzzwords
* Marketing clichés

---

# Hero Copy Guidelines

The hero should answer:

### What is AbsoluteUI?

### Why should I care?

### Why is it different?

Within seconds.

---

# Component Description Guidelines

Every component description should follow:

### Format

What it is.

How it works.

Why it exists.

---

## Good Example

"Interactive card component featuring layered depth, spotlight effects, and smooth cursor-aware motion."

---

## Bad Example

"A stunning next-generation card that transforms your user experience."

---

# Installation Content

Installation instructions should be:

* Direct
* Simple
* Actionable

Avoid unnecessary explanations.

Users came to install the component.

Help them do that quickly.

---

# Documentation Philosophy

Documentation should prioritize:

1. Preview
2. Installation
3. Usage
4. Customization
5. Technical Details

The preview always comes first.

---

# Component Naming Guidelines

Names should feel intentional.

Preferred:

* Aurora Button
* Drift Image
* Eclipse Tooltip
* Horizon Card
* Orbit Navigation

Avoid:

* Button 01
* Animated Button
* Premium Card
* Fancy Tooltip

Names should be memorable.

---

# Code Comments

Comments should explain:

* Why something exists
* Why a decision was made

Avoid:

Explaining obvious code.

Bad:

```js
// Increment count by 1
count++;
```

Good:

```js
// Prevent animation state desynchronization
```

---

# Release Notes

Release notes should focus on:

* What changed
* Why it changed
* Impact

Avoid:

Marketing language.

---

# GitHub Writing Guidelines

GitHub content should feel:

* Professional
* Technical
* Easy to scan

Users should understand the project quickly.

---

# Social Media Guidelines

Social content should focus on:

* Component showcases
* Motion breakdowns
* Design process
* Technical craftsmanship

Avoid:

* Engagement bait
* Trend chasing
* Generic productivity posts

---

# Forbidden Language

Avoid:

* Revolutionary
* Disruptive
* Next-generation
* Cutting-edge
* Future-proof
* Best-in-class
* Ultimate
* Industry-leading

These words reduce credibility.

---

# Preferred Language

Use:

* Crafted
* Curated
* Refined
* Intentional
* Immersive
* Motion-driven
* Production-ready
* Accessible
* Performant

These align with the AbsoluteUI identity.

---

# AI Content Policy

AI can assist writing.

AI should not define the voice.

Before publishing content ask:

* Does this sound human?
* Does this sound specific?
* Does this sound useful?

If not, revise it.

---

# Documentation Quality Checklist

Before publishing:

* Is it clear?
* Is it concise?
* Is it useful?
* Is it specific?
* Is it easy to scan?

If any answer is "No", improve it.

---

# Long-Term Content Goal

The best content feels invisible.

Users should focus on the components, not the writing.

Content exists to create clarity.

Nothing more.

Nothing less.

---

# AbsoluteUI Content Standard

Every sentence should earn its place.

If a sentence does not improve understanding, remove it.
