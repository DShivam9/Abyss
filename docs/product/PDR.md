# Product Requirements Document (PDR)

# Project

AbsoluteUI

---

# Vision

AbsoluteUI is a premium open-source collection of motion-crafted React components designed to transform ordinary websites into immersive digital experiences.

The goal is not to become the largest component library.

The goal is to become the most visually memorable experiential component platform.

Every component should feel handcrafted, polished, intentional, and worthy of being showcased on a modern product website.

The website itself is the first component — a curated digital exhibition demonstrating scroll-driven storytelling, atmospheric depth, and coordinated motion.

---

# Mission

Help developers create visually exceptional websites through beautifully designed components, immersive motion systems, and production-ready experiences.

---

# Product Positioning

AbsoluteUI is not a utility-first component library.

AbsoluteUI is a beauty-first experiential component platform.

Where other libraries focus on quantity and coverage, AbsoluteUI focuses on craftsmanship, presentation, and immersive scroll-driven narratives.

# User Goals

Users should be able to:

* Discover premium components instantly (taking inspiration from 21st.dev's discoverability and organization systems)
* Preview complex motion interactions smoothly in high-fidelity preview environments
* Copy code and install components effortlessly (via CLI or copy-paste)
* Customize components easily
* Improve the visual quality of projects

# Experience Goals

When users visit AbsoluteUI they should feel:

* Inspired
* Impressed
* Curious
* Confident
* Immersed

The website itself should demonstrate the quality of the library through progressive scroll-driven discovery:

1. Hero — Immediate visual impact and interactivity
2. Philosophy — Editorial statement of design intent
3. BentoGrid — Interactive token and component demonstrations
4. Manifesto — Hands-on motion playground
5. Showcase — Pinned editorial component gallery
6. Atmosphere — Canvas-based environmental depth
7. Compositions — Coordinated page layout demonstrations

# Non Goals

AbsoluteUI is NOT intended to become:

* Enterprise Dashboard Framework
* Form Library
* Data Table Library
* CRUD Toolkit
* Generic UI Framework

The focus remains visual excellence and motion craftsmanship.

---

# Guiding Principle

Quality over quantity.

A component should exist only if AbsoluteUI can build a noticeably better version than what already exists.
